# 选择器

选择器（Selector）是一组规则，通过检查文件、文件夹和环境条件来自动检测你的项目类型。匹配成功后，它会添加技能或安装技能集。

## 概述

一个选择器包含：
- **名称** - 显示名称（如"Rust 项目"）
- **规则** - 一个或多个检测条件
- **匹配模式** - 规则的组合方式（全部匹配、任一匹配或自定义表达式）
- **操作** - 要添加的技能和要安装的技能集
- **优先级** - 优先级高的选择器优先生效

## CLI 命令

```bash
# 列出所有已配置的选择器
savhub selector list

# 查看选择器详情（支持部分名称匹配）
savhub selector show rust

# 对当前目录测试选择器（不做修改）
savhub selector test
```

`savhub selector` 命令也可以通过 `savhub detector` 使用（向后兼容）。

## 规则类型

| 规则 | 说明 | 示例 |
|------|------|------|
| **文件存在** | 检查文件是否存在 | `Cargo.toml` |
| **文件夹存在** | 检查文件夹是否存在 | `src/` |
| **Glob 匹配** | 使用 glob 模式匹配文件 | `**/*.rs` |
| **文件包含** | 检查文件是否包含子字符串 | `path: package.json`, `contains: react` |
| **文件正则** | 使用正则表达式匹配文件内容 | `path: Cargo.toml`, `pattern: salvo` |
| **环境变量** | 检查环境变量是否已设置 | `RUST_LOG` |
| **命令执行** | 检查命令是否以 0 退出 | `which cargo` |

所有文件/文件夹路径相对于选择器的**文件夹范围**（默认为 `.`，即项目根目录）。

## 匹配模式

### 全部匹配
所有规则都必须为真（AND 逻辑），这是默认模式。

### 任一匹配
至少一条规则为真（OR 逻辑）。

### 自定义表达式
使用规则编号（从 1 开始）编写布尔表达式：

```
(1 && 2) || !3
```

支持的运算符：
- `&&` - 与（AND）
- `||` - 或（OR）
- `!` - 非（NOT）
- `()` - 分组

示例：一个有 3 条规则的选择器使用 `(1 && 2) || 3` 表示"规则 1 和 2 必须同时匹配，或者仅规则 3 匹配即可"。

## 操作

当选择器匹配时，可以：

1. **添加技能** - 直接按 slug 添加单个技能
2. **添加技能集** - 安装整个技能集合

## 优先级

选择器有一个整数优先级（默认为 0），值越大越先执行。当多个选择器匹配时：
- 所有匹配的选择器贡献的技能和技能集取并集
- 合并后的操作一起执行

## 默认选择器

首次使用时，Savhub 会为常见项目类型创建一组默认选择器，你可以在桌面应用中编辑或删除它们，也可以直接编辑 `selectors.json` 文件。

## 管理选择器

推荐通过 **Savhub 桌面应用**管理选择器，它提供可视化编辑器：
- 规则构建器 UI
- 表达式编辑器
- 技能/技能集搜索
- 模板支持

选择器存储在 `~/.config/savhub/selectors.json`。如果存在旧版 `detectors.json` 文件，会自动迁移。

## 选择器示例

一个 Rust 项目选择器：

```json
{
  "id": "rust-project",
  "name": "Rust Project",
  "description": "检测 Rust/Cargo 项目",
  "folder_scope": ".",
  "rules": [
    { "kind": "file_exists", "path": "Cargo.toml" }
  ],
  "match_mode": "all_match",
  "custom_expression": "",
  "add_skills": [],
  "add_flocks": ["rust-dev"],
  "priority": 10
}
```
